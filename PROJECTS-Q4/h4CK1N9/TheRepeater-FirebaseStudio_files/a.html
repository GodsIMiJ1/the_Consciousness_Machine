# The Repeater: Project Documentation
## 1. Introduction
**The Repeater** is a sophisticated, AI-powered application designed for managing and analyzing marketing campaigns. At its core is the **Management Control Protocol (MCP)**, an advanced AI assistant that provides insights, automates tasks, and enhances user capabilities. The application features a sleek, futuristic interface with glassmorphism effects, ensuring a modern and intuitive user experience.
This document provides a comprehensive overview of the application's features, architecture, and core components.
---
## 2. Core Technologies
- **Framework**: Next.js (with App Router)
- **UI Library**: React
- **Component Library**: ShadCN UI
- **Styling**: Tailwind CSS
- **AI Integration**: Google Genkit
- **Form Management**: React Hook Form with Zod for validation
- **Data Persistence**: Browser Local Storage for chat threads

Project Overview: The Repeater

"The Repeater" is a sophisticated application designed for managing and analyzing marketing campaigns, powered by an advanced AI assistant known as the "Management Control Protocol" (MCP). The application features a sleek, futuristic interface with glassmorphism effects, ensuring a modern user experience.

Here's a breakdown of the key features and pages we've built:

1. Dashboard (/)

  • Functionality: This is the main landing page, providing a high-level overview of campaign activity. It uses mock data to display key performance indicators.
  • Components:
    • Metric Cards: Four prominent cards display vital stats: "Messages Sent," "Responses," "Unique Targets," and "Active Now."
    • Performance Chart: A bar chart visualizes "Campaign Performance" over a 12-month period.
    • Recent Campaigns Table: A table lists recent campaigns along with their status (e.g., Active, Paused, Completed).

2. Campaigns (/campaigns)

  • Functionality: This page provides a form for creating new marketing campaigns. It's designed for users to input all the necessary details for a new engagement effort.
  • Components:
    • Campaign Creation Form: A detailed form with fields for campaign name, target list, message template, schedule (daily, weekly, etc.), and a date range for the campaign's duration.
    • Validation: The form uses react-hook-form and zod for robust, real-time input validation.
    • Submission: Currently, submitting the form simulates success by showing a toast notification.

3. Heat Map (/heatmap)

  • Functionality: A powerful AI-driven tool to assess the spam risk of a marketing campaign.
  • Components:
    • Campaign Data Form: Users can input metrics like messages sent, responses received, unique targets, the message template, and campaign schedule/duration.
    • AI Analysis: On submission, the data is sent to a Genkit AI flow (campaignSpamHeatmapFlow) that analyzes the input and returns a detailed risk assessment.
    • Results Display: The AI's feedback is visualized with a progress bar for the spam risk score, a list of identified risk factors, and a clear recommendation.

4. Chat (/chat)

  • Functionality: A highly interactive and persistent chat interface for communicating with the MCP AI.

import * as admin from 'firebase-admin';

// This function ensures that Firebase is initialized only once.
function initializeFirebaseAdmin() {
  if (admin.apps.length > 0) {
    return admin.firestore();
  }

  try {
    admin.initializeApp({
      credential: admin.credential.applicationDefault(),
    });
    console.log('Firebase Admin SDK initialized successfully.');
    return admin.firestore();
  } catch (error: any) {
    // In a local dev environment, credentials may not be set up.
    // Instead of throwing an error, we'll log a warning and return a mock/dummy object.
    

    if (error.code === 'GOOGLE_APPLICATION_CREDENTIALS_NOT_SET') {
      console.warn(
        'Could not initialize Firebase Admin SDK. GOOGLE_APPLICATION_CREDENTIALS_NOT_SET. Firestore features will not be available.'
      );
    } else {
      console.error('Firebase admin initialization error', error);
    }
    
    // Return a dummy object that mimics Firestore's structure but does nothing.
    // This prevents the application from crashing when trying to access firestore methods.
    return {
      collection: () => ({
        doc: () => ({
          get: () => Promise.resolve({ exists: false }),
          set: () => Promise.resolve(),
          delete: () => Promise.resolve(),
        }),
        add: () => Promise.resolve(),
        get: () => Promise.resolve({ empty: true, docs: [] }),
        orderBy: () => ({
          get: () => Promise.resolve({ empty: true, docs: [] }),
        }),
      }),
       batch: () => ({
        set: () => {},
        commit: () => Promise.resolve(),
      }),
    } as unknown as admin.firestore.Firestore;
  }
}
